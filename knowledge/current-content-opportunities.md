# Current Content Opportunities

## Main Lane

The open lane is "Claude/Codex/AI systems for creators," not developer tutorials.

Recent YouTube search is dominated by Claude Code/Cowork tutorials for developers and business operators. The report found zero strong creator-framed winners for "Claude Code for creators," while "Claude + Nano Banana" was the closest match to Jason's anti-LARP system positioning.

## Highest-Leverage Angles

1. Claude/Codex content systems for creators

Use the exact gap: everyone is teaching Claude Code to developers; almost nobody is showing creators how to use agents to research, script, edit, post, and grow.

2. Build-in-public content system demos

Mino's "things im doing w AI in my biz rn" worked because it named specific systems and data sources. Jason's repo is literally that: research, scriptwriting, rough cuts, captions, b-roll, posting.

3. Comment-keyword resource reels

Nick/Chase keep winning with "Comment X to get..." tied to Claude guides, agentic OS, or free tools. Jason's natural version is giving away a concrete skill, prompt, workflow, or repo template.

4. Time-bound challenge videos

Mino's "make a viral reel in 1 hour" is the top long-form pattern. Jason can run this as "I used Codex to turn one raw idea into a posted reel in 60 minutes" or "I built an AI content system in one sitting."

5. Anti-manual workflow rants

The latest rough cut `C1820` already fits this: "Are you still posting manually from the app in 2026?" This should become a recurring series: manual posting, manual research, manual scripting, manual editing, manual analytics.

## Avoid

- Generic AI tool listicles unless there is a system or proof demo.
- Developer-only Claude Code content unless it is clearly translated for creators.
- Obsidian/RAG setup content unless framed as "I skipped the busywork and built the useful part."
- Pure newsjacking without a lead magnet or practical output.
